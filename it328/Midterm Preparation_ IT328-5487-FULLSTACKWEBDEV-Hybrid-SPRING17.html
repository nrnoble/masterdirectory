    Midterm Preparation

    Overview

    Our midterm will be an in-class exam:

    • 1 hour
    • The test may consist of multiple choice, true/false, fill in the blank, and short answer questions
    • You will be asked to write code segments. Remember to be careful with your syntax!
    • The midterm will be worth 40 points

    In preparation for the midterm, please work through the Study Guide below and verify your answers with your peers. Start studying early!

     

    Study Guide

    Note that this study guide is just that -- a guide. It is not comprehensive, but is intended to give you an idea of the types of things you need to know for the midterm.

    Review

    1. Explain how the bootstrap grid arranges elements. Describe how containers, rows and columns work together to achieve a consistent layout.
    2. A web developer can post a form back to the same page that contains the form definition. Why is this useful in some cases?
    3. What are sticky forms and why are they important?
    4. How do you redirect a user to another page in a PHP script?
    5. What happens when text is sent to the client browser before a redirect is attempted? Explain why this makes sense.
    6. Name the four types of statements you can use to include files in a PHP file, and describe the difference between them.
    7. What two super-globals are used to store the results of a form submission? Which one is more secure, and why?
    8. What function must be called before interacting with session data on any PHP page?
    9. How do you create a session variable in PHP?


    Object Oriented Programming

    1. How do we name a constructor in PHP?
    2. Write a PHP class called Student which stores sid, name, and gpa variables. Then write a parameterized constructor for the class.
    3. What visibility modifiers are supported in Php classes for fields and methods, and what is the difference between them?
    4. Can a PHP constructor support default values? If so, how would you modify your Student constructor to support default values?
    5. What is method overloading? Does PHP support this?
    6. What is method overriding? Does PHP support this?
    7. What is wrong with the following code segment?
    <?php
       class Color
       {
          private $red;
          private $green;
          private $blue;
          public function __construct($r, $g, $b)
          {
             $red = $r;
             $green = $g;
             $blue = $b;
          }
       }
    ?>
    1. Can a Php class can have both a default and parameterized constructor?
    2. Write a single line of codethat invokes a parent class constructor in a child class constructor.
    3. Write a set method and a get method for the Student's gpa.
    4. Write an example toString() method for the Student class, above.
    5. Write code to:
      1. Create a Student object
      2. Set the student's gpa
      3. Print the student, using the toString() method 


    Php Data Objects (PDO)

    1. What does PDO stand for?
    2. What is the difference between PDO and mysqli?
    3. What information is necessary to instantiate a PDO object and connect to a remote database?
    4. Give an example code segment that handles exceptions that are thrown when instantiating the PDO class.
    5. Suppose you were given a PDO object with a connection to a remote database. Given the following query, write a loop that displays the contents of each row in the result set.
    $select = ‘SELECT rank, suit FROM cards WHERE type=”poker”;
     $results = $pdo->query($select);
     
    //print the cards here…
    1. Which constant from the PDO class do we use to retrieve a row of results from the database as an associative array?
    2. Which is the difference between the fetch() and fetchAll() methods, which can be called on a PDO or PDOStatement object?
    3. Prepared statements typically include named placeholders which can accept inputs to a query. Name two functions that can be used to embed these inputs into a query with PDO.
    4. What is SQL injection? Why is it important to understand for web developers?

     

    Code Standards & DocBlocks

    1. State the coding standards for each of the following:
      1. Maximum characters for a line of code
      2. Brace placement for if-statements, loops, function and class definitions
      3. File names
      4. Naming conventions for classes, fields (including private), functions, variables and constants
    2. Name each of the elements that are required for a file-level DocBlock, class-level DocBlock and function DocBlock.
      1. What tags does each DocBlock require?
    3. Which tool do we use to generate a website based upon our DocBlocks?
    4. Give an example @author tag for a file you might write.


    Package Managers

    1. What is Composer and why is it useful?
    2. How does Composer manage packages that a project needs access to?
    3. Can you request specific versions for a package? For example, you might not want the latest build for a framework you are using.
    4. What website gives you access to packages available through Composer?
    5. What commands do we use with Composer to install or update our project packages?
    6. What is the file that we need in our project that stores our configuration information, and what should that file contain?


    Linux

    1. What command do we use to:
      1. Display our current directory?
      2. Display the directory contents?
      3. Move up a directory?
      4. Move to our home directory?
      5. Create a directory?


    Fat-Free

    1. What is Fat-Free?
    2. Describe the steps in creating a new Fat-Free project.
    3. What components must be in our controller (index.php) for a Fat-Free project?
    4. How do we define a default route in Fat-Free?
    5. How do we define an additional route in Fat-Free?
    6. What file do we need to enable routing in our projects?
    7. How do we redirect the user to a different page in Fat-Free?
    8. Write the code to render a view in Fat-Free.
    9. How do we create Fat-Free variables? Write a statement that defines an F3 variable.
    10. How do we handle parameters in Fat-Free? Write a route "hello" that accepts a name parameter, and prints "Hello, name"
    11. When we want to use a new PHP class in our project, what are the steps?

     

    Source Control

    1. Write each of the steps in the Git work-flow we discussed in class.
      • Note: There are at least five steps in the work-flow.
    2. Explain (clearly) the difference between a working copy and a local repository.
    3. What does it mean that Git is a distributed version control system, as opposed to a centralized version control system?
    4. Suppose you create a new file called my_page.php in your working copy. Show the three Git statements you would need to execute to stage your new file, save your changes to your local repository and then save the same changes to your remote repository.
    5. Name the three states that a file can be in with Git.
    6. Which Git command do we use to view the state of our working copy and the Git’s staging area
    7. Which Git command do we use to track commits made to a repository over time?
    8. Name a few good (and bad) habits a developer can have when interacting with a version control system.
