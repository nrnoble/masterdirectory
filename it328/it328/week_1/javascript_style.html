• Always use semicolons at the end of each statement
  • Use upper case for constant names: SOME_CONST
  • Avoid using keyword const because its not supported in all browsers
  • Function names use camel case: SomeFunction
  • Variable names are the same as Java: someVariableName
  • Use single quote rather than double quotes to reduce problems with PHP
  • Don't use global variabiles
  • Declare local variables at the top of scope and initialize
  • Include a single space between operators
  • Use 4 space for code indentation
  • Put opening curly brace at the end of first line of statement
  • Use multiple lines when for each property definition of an object
  • Lines should be less than 80 characters wide
  • Do not use $ in variable names, function names, or object names.
  • Use lower case for all = file names because unix\linux file names are case sensitive
  • {} vs new Object()
  • 0 vs new Number()
  • false vs new Boolean()<= /span>
  • [] vs new Array()
  • /()/ vs new RegExp()
  • function (){} vs new Function()
